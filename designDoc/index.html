List Manager Design Document

Randy Seltzer, Yumeng Sun

Short Summary:

An application that is intended to provide a structured lists of items(tasks) you need to accomplish, events you need to attend, or keyword you need to research on.

Audience:

The intended audience for this App is someone who feels that they would like to help better organize their own personal “To Do” list. We are mainly targeting the general public with this application due to its simplicity but yet very effective use in everyday lives. If we were to go more into depth with the project we could consider adding more functionality to better suit students.

Goals of Use:

The goal for the user is to be adequately reminded of certain chores/activities they add to the reminder list of the app. The goal is to put the user’s mind at ease when it comes to scheduling and giving a sense of reassurance to the user that they will be reminded of that important activity through a push notification and/or email.

Comparison to other list/reminder apps (Competitive Analysis):

  • ColorNote Notepad Notes: native on Android, integrated calendar and categorized lists into different categories, color coded the categories and tasks.
  • Memo Pad: native on Android, in Japanese, provides search function, can show up as widget

Our app is very bare minimum with functionality so it allows for all users to comfortably work through the app without getting confused or lost. Some of the other apps we found have a social networking aspect to it, which we thought was a good idea, but the way the social portion of their app was explained was that is delegated responsibility and assigned tasks to other users. That kind of functionality seemed to really target managers and higher level employees who actually have someone working under them which was not what we intend for our users. Some apps like Astrid send push notifications and emails as their “reminder” and we liked the idea of that so we decided that we are going to try and implement that into our interface.

Data Storage:

We plan to store data locally so that the user will not lose their data each time they close the app. We would like for their list to remain “loaded” each time they decide to use the app, by utilizing localstorage.

Interface Features:

  • A list box which stores the user’s “chores”
  • Check boxes to provide the option of a push notification
  • text field which has a small description of the action to be reminded of
  • A text field for the date
  • A text area for a more in depth description of the activity
  • A button which adds the strings from the two text fields and the text area to the currently existing list of reminders
  • A possible checkbox option for a daily reminder

Usability Requirement Benchmark:

  • Average user can record a line of note and push that into the top of memo in 1 minute
  • Display to do list in 3 clicks

Application Architecture:

Web app.

Object oriented application. Provide simplicity and be prepared for more improvement and future development.

Key Object:

List (consists of 0 or many entries)

Entry (have attributes: date, description, doPushNotification, isDone, priority, type)
